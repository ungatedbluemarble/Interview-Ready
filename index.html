What This Is

Interview Ready is a structured, research-grounded career preparation system built as Claude skills. Each skill is autonomous but context-aware: they share a user profile so you are never asked the same question twice across the workflow.

Every output is grounded in actual company research and your real background. This is not a template generator.

Works in Claude.ai (browser), Claude Desktop, and Claude Code. Gemini Gem port planned

The Four Skills

SKILL 01
Interviewer

Onboards the user. Builds the shared profile: background, strengths, weaknesses, job search status and motivation. The starting point for every other skill.

SKILL 02
Resume Writer

Accepts your resume via file upload or any document source. Rewrites or builds your resume and cover letter in professional, human language. Primary output is Word (.docx).

SKILL 03
Job Search Expert

Matches your profile to industries and roles. Evaluates job descriptions you bring: individually or in bulk. Tells you whether to apply and how to position.

SKILL 04
Interview Prep

Prepares you for every stage of the hiring pipeline: recruiter screen, hiring manager, panel, final round, and offer negotiation. Research-grounded, role-specific prep documents with live salary law and compensation intelligence.

Requirements

  • Claude.ai account (Pro, Max, Team, or Enterprise) or Claude Code
  • Code Execution enabled: Settings → Capabilities → Code Execution (required for skills to load)
  • No API key required. You bring your own Claude environment.

Installation

Option 1: Claude.ai or Claude Desktop (no technical setup required)

This is the path for most users.

1
Download the package

On the GitHub page, click the green Code button, then Download ZIP. Save the file to your computer.

2
Open Claude settings

In Claude.ai: click your profile icon top right → Settings → Features → Skills.
In Claude Desktop: Settings → Capabilities → Skills.

3
Enable Code Execution

In Settings → Capabilities, make sure Code Execution is turned on. Skills will not load without it.

4
Upload each skill

Each skill is a separate folder inside the ZIP. Create a ZIP for each folder and upload one at a time using the Upload skill button.

On Mac: right-click the skill folder → Compress
On Windows: right-click → Send to → Compressed (zipped) folder

Option 2: Claude Code

git clone https://github.com/ungatedbluemarble/interview-ready.git
cp -r interview-ready/skill-* ~/.claude/skills/

Option 3: One-liner

mkdir -p ~/.claude/skills && \
  git clone https://github.com/ungatedbluemarble/interview-ready.git /tmp/interview-ready && \
  cp -r /tmp/interview-ready/skill-* ~/.claude/skills/

Usage

Once installed, start with Skill 01. Everything else follows from there. In any Claude session, say:

"I'm ready to start Interview Ready."

or

"I need help with my job search."

Skill 01 will onboard you, build your profile, and route you to the right skill based on what you need.

Skill Trigger Reference

What to saySkill triggered
"Start Interview Ready" / "I need help with my job search"Skill 01: Interviewer
"Write my resume" / "Update my cover letter"Skill 02: Resume Writer
"Find jobs for me" / "Review this job description"Skill 03: Job Search Expert
"I have an interview" / "Prep me for my interview"Skill 04: Interview Prep
"Show me my pipeline status" / "Where am I with my applications"Skill 04: Pipeline Dashboard

Bringing In Your Documents

You can feed your resume and other career documents from any source:

  • Direct upload: drag and drop a file into the Claude chat window
  • Copy and paste: paste the text of any document directly into chat
  • Cloud storage: Google Drive, OneDrive, Dropbox, Box, SharePoint, or any connected service
  • Local files: in Claude Code or Claude Desktop, reference files anywhere on your computer
  • Any format: PDF, Word (.docx), plain text (.txt), or pasted text

How Context Works

Session-persistent

The profile lives in the active conversation. No file is written unless you request it. Say save my profile to export.

File-based

Export your profile at any time. Upload interview_ready_profile.json in future sessions to continue without re-interviewing.

Managing Your Session

Interview Ready is designed to be used across multiple sessions, not in one sitting. Each skill is token-intensive and running the full pipeline end to end in a single conversation will approach Claude's context limits.

Estimated Token Usage

StageEstimated Tokens
Skill 01: Intake and profile build2,000 to 4,000
Skill 02: Resume rewrite and cover letter8,000 to 15,000
Skill 03: Single JD evaluation3,000 to 5,000
Skill 03: Bulk JD review (5 JDs)10,000 to 15,000
Skill 04: Single stage prep document8,000 to 12,000
Skill 04: Mock interview session4,000 to 8,000

A complete end-to-end run consumes roughly 35,000 to 55,000 tokens. Run one skill per session and save your profile after each one.

Recommended Session Breakdown

Session 1: Skill 01 intake and profile build. Save profile.

Session 2: Skill 02 resume and cover letter. Save profile.

Session 3: Skill 03 job search and JD evaluation. Save profile.

Session 4: Skill 04 recruiter screen prep. Mock interview. Save profile.

Session 5: Skill 04 hiring manager prep. Continue through final round and offer negotiation.

Built-in Intelligence

Live Salary Law Search

Skill 04 includes a live salary law search that runs on first use and refreshes automatically every 90 days. When you reach the compensation section of any recruiter screen prep, the skill searches for current salary history and expectation ban laws in your state and city across five authoritative sources:

SourceCoverage
State labor department websitesOfficial state-level salary history and expectation ban laws
National Conference of State Legislatures (ncsl.org)Tracks salary history ban legislation by state, updated regularly
U.S. Department of Labor (dol.gov)Federal wage and hour guidance and pay equity resources
EEOC (eeoc.gov)Pay equity and employment discrimination law guidance
Ballotpedia (ballotpedia.org)Recent legislative changes and ballot measures by state

The result is always accompanied by a disclaimer directing you to your state Department of Labor to verify. Laws change. The skill makes this clear every time.

Live Compensation Benchmarking

Pipeline Dashboard

Your exported profile contains an interview_history field that tracks every company, role, pipeline stage, and prep session. Ask Claude at any point:

"Show me the status of all my active applications."

Troubleshooting

Skills are not triggering: Make sure Code Execution is enabled in Settings → Capabilities. Skills will not load without it.

Document upload is not working: Any format Claude supports works: PDF, Word, or plain text. If a file fails to upload, paste the text directly into chat instead.

Session ran out of context: Say "save my profile" immediately, start a new session, upload the profile file, and continue.

A Note on LinkedIn

LinkedIn blocks automated content fetching. When this package researches a hiring manager, it extracts their name and company from the URL you provide and runs web search queries instead, synthesizing results from public sources. If minimal public data exists for an interviewer, the skill flags it and focuses on role-level signals instead.
